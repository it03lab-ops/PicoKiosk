# PicoKiosk

**Лаунчер для VR‑шлема PICO 4 в режиме киоска**  
Форк [PicoZen](https://github.com/barnabwhy/PicoZen) c доработками под публичные сценарии: выставки, шоурумы, обучение, демо‑стенды и т.п. [page:0]

> ⚠️ Неофициальный проект, не связан с PICO или Meta.  
> ✅ Предназначен для шлемов, которыми пользуются **много разных людей**.  
> 🔒 Ограничивает пользователя только разрешёнными приложениями, блокирует доступ к системным настройкам и стандартному лаунчеру.

Обычно PicoKiosk используется **в связке** с утилитой  
[`pico4-adb-debloat`](https://github.com/it03lab-ops/pico4-adb-debloat),  
которая готовит шлем Pico 4 к киоск‑режиму: отключает лишние приложения, блокирует магазин и системный UI, применяет твики и устанавливает нужные APK через ADB. [page:1]

---

## Зачем два проекта?

PicoKiosk и `pico4-adb-debloat` закрывают разные уровни:

| Уровень                 | Инструмент               | Задача |
|-------------------------|--------------------------|--------|
| Система / прошивка      | `pico4-adb-debloat`      | Очистка и «жёсткая» настройка шлема через ADB: отключение bloatware, блокировка магазина и настроек, твики, массовая установка APK и перенос конфигурации между шлемами. [page:1] |
| Приложения / UI         | PicoKiosk (этот репо)    | Замена стандартного лаунчера на киоск‑меню: только белый список приложений, скрытие системного UI, защита настроек паролем. [page:0] |

PicoKiosk можно использовать отдельно, но для **настоящего киоск‑режима на Pico 4** настоятельно рекомендуется:  
сначала прогнать `pico4-adb-debloat`, затем установить и настроить PicoKiosk. [page:0][page:1]

## 🔗 Companion‑утилита: pico4-adb-debloat

[`pico4-adb-debloat`](https://github.com/it03lab-ops/pico4-adb-debloat) — это batch‑скрипт для Windows, который управляет шлемом Pico 4 по ADB. [page:1]

Основные возможности:

- Отключение/включение системных и вендорских пакетов (`pm disable-user` / `pm enable`). [page:1]  
- Разбиение на логические блоки:
  - безопасно отключаемые сервисы (телеметрия, демо‑контент и пр.);
  - core‑приложения (лаунчер, браузер, TOB‑софт и т.п.);
  - опциональные пакеты и твики производительности/сети;
  - блок **«Kiosk»** – закрытие магазина, настроек, файловых диалогов и других «выходов» из киоска. [page:1]
- Работа с конфигами:
  - сохранение текущего состояния пакетов в `pico4_config.txt`;
  - применение того же конфига на других шлемах. [page:1]
- Простая установка APK:
  - сканирование папки `apk/`;
  - установка одного или всех APK через `adb install` / `adb install -r`. [page:1]

Использование этого скрипта **до** развёртывания PicoKiosk позволяет:

- убрать лишний мусор из системного UI;
- отключить стоковый магазин и лаунчеры, через которые можно «убежать» из киоска;
- тиражировать один и тот же жёсткий профиль на несколько шлемов.  

Подробности — в README репозитория [`pico4-adb-debloat`](https://github.com/it03lab-ops/pico4-adb-debloat). [page:1]

## 🔧 Рекомендуемый сценарий развёртывания (Pico 4)

1. **Подготовить шлем через `pico4-adb-debloat`:** [page:1]  
   - включить режим разработчика и USB‑отладку на шлеме;
   - подключить Pico 4 к ПК с Windows и ADB;
   - запустить скрипт, пройти по разделам:
     - отключить ненужные/опасные для киоска приложения;
     - включить блоки, относящиеся к киоск‑режиму;
     - установить нужные APK (ваше приложение, PicoKiosk, вспомогательные утилиты).

2. **Установить и настроить PicoKiosk:** [page:0]  
   - собрать или установить APK PicoKiosk;
   - запустить PicoKiosk из системного меню;
   - в настройках (они защищены паролем):
     - задать/изменить пароль администратора;
     - выбрать приложения, которые будут доступны пользователям;
     - включить автозапуск, режим редактирования и другие опции.

3. **Назначить PicoKiosk лаунчером по умолчанию:**  
   - через ADB назначить PicoKiosk в качестве home‑активити, например:  
     `adb shell pm set-home-activity com.yourname.picokiosk/.MainActivity`  
   - проверить, что кнопка Home/системная всегда возвращает в PicoKiosk, а не в стоковый лаунчер.

4. **Протестировать возможные «выходы»:**  
   - шторка уведомлений, быстрые настройки, магазин, системные диалоги и т.п.;  
   - при необходимости:
     - донастроить `pico4-adb-debloat` (отключить дополнительные пакеты),
     - подправить whitelist/hidden в самом PicoKiosk.

## Совместимость и ограничения

- **Основная цель:** Pico 4 (тестировалось на Pico 4). Возможна работа на Neo 3, но не гарантируется. [page:0]  
- **Meta Quest (2/3/Pro):** не тестировался; теоретически должен запускаться, но блокировка системного UI ограничена особенностями VROS. Использовать на свой риск. [page:0]  
- Минимальная версия Android — типичная для современных XR‑шлемов (Android 10+). [page:0]  
- `pico4-adb-debloat` рассчитан конкретно на стоковую прошивку Pico 4 — не используйте его на других устройствах. [page:1]

## Установка PicoKiosk

1. **Сборка APK**

   ```bash
   git clone https://github.com/it03lab-ops/PicoKiosk.git
   cd PicoKiosk
   # Открыть проект в Android Studio
   # Sync Gradle, затем Build -> Build APK(s)
   ```

   Готовый APK окажется в `app/build/outputs/apk/…`. [page:0]

2. **Установка на шлем**

   ```bash
   adb install PicoKiosk-release.apk
   ```

3. **Первый запуск и настройка** — см. раздел «Рекомендуемый сценарий развёртывания».

## Отношение к PicoZen

PicoKiosk — это форк [PicoZen](https://github.com/barnabwhy/PicoZen), который изначально задумывался как универсальный лаунчер. [page:0]  
В PicoKiosk сделан упор именно на **режим киоска**:

- убраны/спрятаны функции, не нужные в публичных стендах;
- добавлен пароль на настройки и на просмотр скрытых приложений;
- улучшена интеграция с ADB‑скриптом `pico4-adb-debloat`. [page:0][page:1]
